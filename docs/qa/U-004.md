## 3. Findings (both non-blocking; both on the U-004 preview chrome that U-012 replaces per ADR-010)

### F-1 (MEDIUM, advisory) — `copySelection()` doesn't trigger a re-render, so the Paste button stays disabled after Copy
`ToolSession.copySelection()` (`src/state/toolSession.ts:290`) sets
`this.clipboard` but — unlike `cut()` and `paste()` — never calls
`this.onChange()`. The React snapshot's `canPaste` (`= session.hasClipboard()`)
therefore does not refresh, and the Paste toolbar button
(`src/ui/CanvasStage.tsx:538`, `disabled={!snap?.canPaste}`) remains **disabled**
after a Copy until some unrelated state change re-renders. Verified on a fresh
session: select → click **Copy** → Paste still disabled (cannot be clicked);
toggling any modifier (which fires `onChange`) then enables it — proving the
clipboard *was* populated and only the re-render was missing.
- **Impact:** the **button-only** copy→paste workflow is broken (a mouse-only
  user cannot reach Paste after Copy). The **keyboard** path (Ctrl+C then Ctrl+V)
  is unaffected because `paste()` runs directly and does not depend on the
  button's disabled state — which is why the acceptance criterion still passes.
- **Steps to reproduce:** fresh load → Select → drag a marquee → click the
  **Copy** button → observe the **Paste** button is still greyed out.
- **Fix:** add `this.onChange();` at the end of `copySelection()` (one line), so
  the Paste/Stamp affordances reflect clipboard state immediately.

### F-2 (LOW, advisory) — wheel zoom updates the viewport but not the Zoom% readout
The wheel handler (`src/ui/CanvasStage.tsx:133` `onWheel`) calls
`renderer.setViewport(zoomAt(...))` but never calls `setZoomPct`, so the status
bar's `Zoom %` stays stale after a wheel zoom even though the canvas actually
zooms (measured: a fixed 160 px screen span covers 10 art px at Fit → 7 after a
wheel-in → 10 after wheel-out). Button zoom is unaffected (its handlers call
`setZoomPct`). Preview-only chrome; fold into U-012's real view controls.
- **Fix:** have `onWheel` push the new zoom into `setZoomPct` (or subscribe the
  status readout to the renderer's viewport).
