## Acceptance criteria — definition-of-done checklist

### Machine-checkable (criteria.md) — independently re-verified live + by suites

1. **Adding a layer adds an aligned layer to EVERY frame.** ✅
   Live: `+ Layer (all frames)` moved the status readout `2 layers → 3 layers`;
   undo restored `2 layers`. Unit/browser tests assert every frame gains the layer
   at the same index with the **same aligned id** but **distinct buffer instances**
   (painting one frame's copy never leaks to siblings).
2. **Add / duplicate / delete / reorder; duplicate deep-copies buffers.** ✅
   Live pixel-truth test on the real app: painted frame 1's Spark layer at art pixel
   (2,2); Duplicate inserted the copy at index 1 and made it active; painted the copy
   at (5,5). Read the two native-resolution timeline thumbnails:
   - copy has (2,2) [alpha 255] **and** its own (5,5) [alpha 255],
   - source keeps (2,2) [alpha 255] and (5,5) stays **empty [alpha 0]**.
   → Editing the copy does **not** mutate the source (true deep copy). Delete guard:
   deleting down to 1 frame left the **Delete button disabled**. All ops undoable.
3. **Playback order + timing derive from per-frame `durationMs` / global `fps`.** ✅
   Live: Play advanced the active frame off frame 0; Pause froze it; Stop returned to
   frame 0; Next/Prev step deterministically; **Ping-pong** advanced forward to frame
   3 then bounced back (`peak=3, sawReturn=true`). Unit tests pin `buildTimeline`
   (cumulative start offsets + total), `frameIndexAtTime` (looping/clamped/ping-pong
   boundaries), `effectiveDurationMs` (per-frame else fps fallback), `cycleDurationMs`.
4. **Onion selection returns only N prev + N next, never current, clamped.** ✅
   Live: on the middle frame, the ghost overlay canvas contained **warm (previous)**
   pixels (`r−b>40`, count 23) **and cool (next)** pixels; setting Prev range = 0
   removed all warm ghosts while keeping cool; ghosts render at **reduced opacity**
   (max alpha 191 < 255). Unit tests assert never-current, asymmetric ranges,
   start/end clamping (no wraparound), zero-range, and `MAX_ONION_RANGE` cap.

## Findings

### F1 — Missing held-out acceptance test for U-008 (LOW, non-blocking, traceability)

`docs/acceptance/U-008/` contains only `criteria.md` — there is **no held-out
`*.test.ts`**. `vite.config.ts` includes the glob
`docs/acceptance/U-008/**/*.test.{ts,tsx}`, but it matches **zero files**
(`npx vitest run docs/acceptance/U-008` → "No test files found"). U-008 is the only
built unit lacking its builder-protected suite (U-003/004/005/006/007/009 each have
one). Consequently the four machine-checkable criteria are validated only by
builder-editable tests (`src/core/frames.test.ts`, `src/state/frameStore.test.ts`).

- **Impact:** the objective gate's held-out-test guarantee is not met for this unit.
  This is a factory acceptance-authoring gap, **not** a builder reward-hack (nothing
  was deleted; `criteria.md` never named a required test artifact) and **not** a
  functional defect — I independently re-verified all four criteria against the live
  app at the pixel level.
- **Fix hint:** author `docs/acceptance/U-008/frames.acceptance.test.ts` (builder-
  protected) importing the pure engine by exact path — mirroring
  `docs/acceptance/U-007/layers.acceptance.test.ts` — asserting: (1) add-layer-to-all-
  frames with distinct buffers + aligned ids; (2) duplicate deep-copy independence +
  delete last-frame guard + reorder; (3) `buildTimeline`/`frameIndexAtTime` order &
  timing incl. ping-pong; (4) `selectOnionFrames` N-prev/N-next never-current +
  clamping. Then re-run the gate so `npm test` exercises the authoritative suite.

### Note — corrected self-check (not a product defect)

My first manual-QA pass reported "focus indicator = none". That was an **invalid
assertion in my own driver** — it read `getComputedStyle(el, ':focus-visible')`
(which treats the arg as a pseudo-*element*) and then the `outline` property, whereas
the design system deliberately uses `outline:none` + `box-shadow: var(--focus-ring)`.
Re-verified under real keyboard focus: a visible two-layer focus ring **is** present
(see `focus-check.log`). No product issue.
