## Findings

### F1 (LOW / informational, non-blocking) ‚Äî layer/frame/resize structural undo not exercisable yet (dependency-graph scoping)
`docs/acceptance/U-006/criteria.md` lists "Structural ops (add/remove/reorder layer &
frame, resize, paste) undo/redo correctly." **Paste (and cut/move/clear/fill/shape)
are wired and verified live**; **layer, frame, and resize** operations cannot be
exercised through the app because the **Layers panel (U-007), Animation frames
(U-008), and Resize dialog (U-011) are not built yet** (all `pending`).

- **Why non-blocking:** the *reversible primitive* those units will use ‚Äî
  `applyListEdit` (insert/remove/move/replace with exact inverse) ‚Äî is implemented in
  `src/core/history.ts` and **unit-tested with 8 cases incl. immutability and edge
  conditions**, and the pixel-patch/command machinery is fully proven. The
  authoritative machine-checkable held-out acceptance requires none of
  layer/frame/resize, and passes 5/5. This is the same dependency-ordered scoping the
  factory applied to U-009's "all-frames" export.
- **Fix / follow-up:** re-verify layer-op undo in **U-007**, frame-op undo in **U-008**,
  and resize/crop/trim undo in **U-011**, each of which depends on U-006 and will wire
  its structural commands onto `applyListEdit` / `replaceBufferWithHistory`.

### Advisory (non-blocking, informational)
- **A-1 (‚Üí U-013 perf pass):** `index` bundle is 279 kB raw / 88.6 kB gz ‚Äî under the
  300 kB gz web budget; track as bundle creep and code-split the export path when the
  app shell lands.
- **A-2 (‚Üí U-012):** the running preview canvas is 32√ó32 (throwaway per ADR-010); the
  "1px @512¬≤ stores only its sub-rect" guarantee is proven at the pure-engine level
  (patch < 1 KB), not in the live preview. Re-confirm live when the 512¬≤-capable
  workbench arrives in U-012.
- **A-3 (INFO):** byte-cap eviction is exercised at unit level only (a live 64 MiB run
  is impractical); the running depth cap was exercised live (exactly 100). Adequate.

No other defects found. Every machine-checkable acceptance item and every relevant
test type (smoke, functional, regression, integration, accessibility, performance,
stress/boundary, security smoke) passes on real runs.
