## 3. Functional walkthrough — isolated Chromium vs. live dev server (`:5190`)

Clean `localStorage` each run; every value below is the **observed** actual.

**Defaults (screenshot `U-005-01-initial-clean.png`):** fg `#000000`, bg `#FFFFFF`,
active palette **PICO-8 / 16 swatches**, **Indexed ○ (aria-pressed=false) ⇒
free-color default confirmed**, recents empty, editing **foreground**, no lock
readout.

**fg/bg + swap/reset:** palette click (idx 8) → fg `#FF004D`; right-click (idx 11)
→ bg `#00E436`; **Swap** button exchanges → fg `#00E436` / bg `#FF004D`; **Reset**
→ `#000000` / `#FFFFFF`. **Keyboard `X`** (on a focused non-input control) swaps;
**`D`** resets — both verified on real key events.

**HSV picker** (`U-005-02-picker-recents.png`), all keyboard-operable
(`role="slider"`):
- **Hex** `#1E90FF` + Enter → fg set + recorded in recents.
- **Hue** `ArrowRight` 210→211, `Shift+ArrowRight` →221 (`aria-valuetext="221 degrees"`); fg `#1E67FF`.
- **SV** `ArrowDown`/`ArrowLeft` → `Saturation 86%, brightness 98%`; fg `#2268FA`.
- **Alpha** `Shift+ArrowLeft` 255→239 → 8-digit **`#2268FAEF`** (fg + hex input).

**Load every built-in with exact swatch counts:** forge **13**, gameboy **4**,
pico8 **16**, nes **55**, cga **16**, c64 **16**, db16 **16** (grid `aria-label`
matches each palette name).

**Import:** `.hex` → 4 colors, *"Imported 4 colors."*, rows `#0F380F/#306230/
#8BAC0F/#9BBC0F`; `.gpl` → 3 colors, name **"Trio Test"**, rows `#0F380F/#306230/
#8BAC0F`, *"Imported 3 colors."*; **garbage** → error *"Not a valid hex color:
…"*, palette **unchanged** (3 swatches), **no throw**.

**Export:** `.hex` → filename `game-boy-dmg.hex`, blob `text/plain;charset=utf-8`,
body exactly the 4 Game Boy hexes (`; Game Boy DMG — exported from PixelForge` +
`#0F380F/#306230/#8BAC0F/#9BBC0F`), status *"Exported HEX."*

**Swatch editing:** `+ Add fg` appends the current fg `#123456`; **Move right**
swaps idx0↔idx1 (`#0F380F`↔`#306230`); **Set** replaces the selected with fg
(`#123456`→`#ABCDEF`); **Remove** deletes the selected. All immutable.

**Indexed / palette-lock — the H-1 fix, reproduced live**
(`U-005-03-indexed-lock-gameboy.png`):
1. Load Game Boy, fg = off-palette `#FF00FF`; draw on canvas in **free** mode ⇒
   display canvas contains `#FF00FF` (magenta really painted).
2. Toggle **Indexed ●** (aria-pressed=true) ⇒ the whole buffer **quantizes** to
   Game Boy; display colors ⊂ `{#0F380F,#306230,#8BAC0F,#9BBC0F}`, **no magenta**;
   panel readout: **"Palette-locked — drawing as #306230"** (magenta's nearest GB).
3. Draw again with fg still `#FF00FF` ⇒ display **stays** within Game Boy (pencil
   paints the snapped `#306230`); off-palette color **cannot** be re-introduced.

**Palette-swap by index** (`U-005-04-paletteswap-cga.png`): while locked, change
Game Boy → CGA ⇒ art recolors **by index**, verified exactly:
`idx0 #0F380F→#000000`, `idx1 #306230→#0000AA`, `idx3 #9BBC0F→#00AAAA`
(`mappingCorrect: true`; every pixel is a CGA color). The readout re-snaps the
pencil to `#FF55FF` (magenta's nearest CGA). This is the *"index i → color at index
i"* contract on the **real canvas**, not a preview.

**Recents:** committing 30 distinct colors **caps at 24** (oldest evicted,
most-recent-first: `#1E0000 … #070000`); re-committing the front color **dedups**
(length stays 24, front unchanged); persisted to `localStorage`
`pixelforge.recentColors.v1` as `#RRGGBBAA`; **survives a full reload** (24 restored,
same order).

**Edge cases / robustness:**
- **Shortcut guard:** with focus in the **hex input**, pressing `x`/`d` does **not**
  swap/reset (fg/bg unchanged); chars enter the draft only. An invalid draft
  (`#FF004Dxd`) **reverts** to the last valid value on blur.
- **Empty palette:** `New` ⇒ grid shows *"Empty palette — add a swatch below."*, 0
  swatches, *"Started a new palette."*; toggling Indexed on an empty palette yields
  a **strict transparent lock** (readout `#00000000`) — nothing drawable, **no crash**.

**Accessibility & responsive:**
- Sliders are `role="slider"` with arrow/Shift handling + `aria-valuemin/max/now/
  text`; swatches are labelled buttons with `aria-pressed`. **Keyboard bg path**
  verified: **Shift+Enter** on focused swatch idx3 sets bg `#008751` (fg unchanged).
- **Visible focus ring** (WCAG 2.4.7): focused swatch `box-shadow` includes
  `rgb(255,46,136) 0 0 0 4px` (the U-002 Spark focus ring) — not a removed outline.
- **No horizontal overflow** at **1440 / 768 / 375 px** (`scrollWidth ≤ innerWidth`
  at each: 1425/753/360). At 375 the full Color panel + 16 swatches remain visible
  and the app stacks cleanly (`U-005-05-responsive-mobile-375.png`), including the
  live palette-swap gem preview (*"PICO-8 → recolored by index"*).

**Console health:** **no app-origin errors or warnings** across the entire session
(initial load, drawing, locking, palette-swap, import/export, 30+ commits, reloads,
resizes). The only console noise is third-party: `meta.json` root 404 and
`browserextension.trustpilot.com` 404 (browser extensions — the app serves under
`/pixel-forge/` and has **zero** references to either in `src`/`dist`), MetaMask
`chrome-extension://…` listener/multiplex warnings, and one `Canvas2D
willReadFrequently` **hint** originating from the QA `getImageData` pixel-probes
(not the app paint path).

**Design quality:** a deliberate retro build — effective default theme is **Arcade
CRT** (computed `--c-ember: #00f0ff`, per spec), pixel typefaces, hard-edged bevels
(0 blur), transparency checker swatches, and the palette-swap gem preview. Not
