## Observations (non-blocking)

1. **Environment flakiness (harness, not the unit):** mid-session `node_modules/.bin`
   was emptied and `vite` vanished (build/dev had worked minutes earlier),
   consistent with a **concurrent factory process churning the same worktree** —
   `.factory.lock` is present and `state.json phase` is `building`. A clean
   `npm ci` restored `.bin` (54 shims) and every check passed again reproducibly
   (`build`/`test` re-confirmed exit 0). Recommend the engine ensure the
   single-writer lock prevents concurrent `npm` operations in a worktree while QA
   runs. Not a defect in U-001's deliverables.
2. **Uncommitted build:** the whole scaffold is untracked in the worktree —
   expected before the integration step; noted for Verifier/Doc.
3. **`favicon.svg` href is relative** in `index.html`; it resolves to
   `/pixel-forge/favicon.svg` under the base path (asset is present in `dist`). Fine.
4. **`data-theme="light"` on `<html>` at runtime** is injected by a browser
   extension in the test profile, not by the app; it is inert (no matching theme
   rule) so the default Arcade CRT `:root` tokens apply. Not an app issue.
5. **Coverage table shows empty rows** while the summary reports 100% — a cosmetic
   Vitest v8 display quirk; thresholds are enforced and met.
