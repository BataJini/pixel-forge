   `include` is still `src/core/**`, so U-002's pure logic (`ui/theme/contrast.ts`,
   `crt.ts`, `themes.ts`, `platform/audio.ts`) is **not counted** in the 80% gate ‚Äî
   although it IS thoroughly unit-tested (61 passing tests). Recommend widening the
   coverage `include` to cover these pure modules when convenient (U-005/U-012).
   Not a U-002 acceptance blocker (no coverage number is required by U-002 criteria).
2. **`data-theme="light"` on `<html>` at runtime (environmental, not the unit):**
   injected by a custom-theme browser extension in the shared MCP Chrome profile
   (also injects inline `--font-family:'Inter'` / `--border-radius:6px`). It is
   inert ‚Äî the app defines no `[data-theme="light"]` rule (only arcade/forge/
   gameboy/amber) and uses none of those vars, so resolution correctly falls back
   to the Arcade `:root` and `--c-ember` stays `#00f0ff`. Same artifact was noted
   in the U-001 report. Confirmed benign in the isolated (extension-free) runs.
3. **Persisted preferences in the shared profile:** the MCP profile carried a prior
   `pixelforge.soundEnabled=true` / `crtLevel=subtle`, briefly showing "SFX ON"
   before I cleared storage. The genuine muted-by-default was re-confirmed on a
   clean profile. Expected persistence behavior, not a defect.
