## Pass 1 — Spec / acceptance conformance

Held-out criteria (`docs/acceptance/U-003/criteria.md`) + work-breakdown U-003 DoD:

1. **Color round-trip** — MET. `src/core/color.ts` `hexToRgba`/`rgbaToHex` lossless for
   8-bit incl. alpha; `#RGB`/`#RRGGBB`/`#RRGGBBAA`; `tryHexToRgba` result-envelope for
   untrusted input, `hexToRgba` throws on programmer error (constitution).
2. **Buffer create/get/set immutability + OOB no-op** — MET. `createBuffer` all-transparent;
   `setPixel` returns a NEW buffer, original untouched; OOB read → `[0,0,0,0]`, OOB write →
   no-op equal buffer. Held-out + unit tests confirm.
3. **Composite alpha/z-order** — MET. Source-over (`overPixel`), z-order bottom→top,
   skips hidden/zero-opacity, opacity 0–100 scales source alpha, transparent reveals below.
   Builder tests add 50%-over-black and two-semitransparent output-alpha cases.
4. **dirtyRect correctness** — MET. Tight bounding box, `null` when identical, whole-`after`
   on size mismatch, 32-bit word fast-path with a byte fallback for unaligned views (tested).
5. **Nearest-neighbor display** — MET. `renderer.ts` sets `imageSmoothingEnabled=false` and
   CSS `image-rendering: pixelated`; browser test asserts no anti-aliased ramp at 40× zoom.
6. **Zoom/pan coordinate mapping** — MET. Pure `viewport.ts` (`screenToArt`/`artToScreen`
   inverse, `zoomAt` cursor-anchored, integer `ZOOM_STEPS`, `fitToScreen`); browser test maps
   pointer→art coords under pan+zoom.
7. **Single pointer op at 512×512 repaints only its dirty rect within one frame** — MET.
   rAF-coalesced, dirty-rect-scoped `updateRegion`/`markDirty`; browser test at the **512×512**
   ceiling proves exactly one 1×1 blit (no ~262k-px full repaint) and union-coalescing.
8. **Checkerboard/grid overlay-only, never in buffer** — MET. `overlays.ts` paints checker on
   the backdrop and grids on the overlay canvas; source buffer stays clean (browser test:
   display alpha 0 on transparent pixels, checker only on backdrop). Structurally satisfies the
   clean-export invariant foundation.

Manual/review items: pixel grid appears only at/above `gridMinZoom`; tile grid optional;
fresh buffer fully transparent — all confirmed. Coordinate mapping correct at all zoom/pan.

## Pass 2 — Engineering / correctness / reward-hack

- **Module boundaries (constitution)** — clean. Pure engine in `src/core/**` (no DOM/React;
  `buffer/color/rect/viewport/types`), browser glue in `src/platform/**` (`renderer`,
  `overlays`), React only in `src/ui/CanvasStage.tsx`. Held-out imports resolve by exact path.
- **Immutability** — honored. `setPixel` copies; the in-place `setPixelMut`/`fillRectMut`
  live in the buffer module, are bounded to a returned dirty rect, and are the only mutation
  path (exactly the constitution's scoped exception). `getPixel` returns fresh tuples.
- **Determinism/purity** — engine ops are pure and DOM-free; render side effects isolated.
- **KISS/DRY/size** — largest files renderer 337 / CanvasStage 333 LoC (< 800). No dead code,
  no `console`/`debugger`, no TODO/FIXME, no `@ts-ignore`/lint-disables.
- **Security** — N/A surface (no auth/files/network/crypto). Color parser validates at the
  boundary; no `eval`, no `innerHTML`; `rgbaToHex` feeds only a CSS `background` value. No
  security-reviewer delegation warranted.
- **Reward-hack scan — CLEAN.** Held-out `docs/acceptance/U-003/**` is **byte-identical to
  master** (diff = 0). No skipped/`.only`/`xfail`/`todo` tests, no weakened/deleted assertions,

## Findings (all advisory — do not block)

- **F-1 (LOW) — Space+drag and two-finger pan not wired in the preview.** `CanvasStage.tsx`
  implements middle/right-drag pan + wheel zoom only; the U-003 criteria manual list also names
  Space+drag and two-finger drag. The underlying viewport math (`panBy`, `screenToArt`) fully
  supports every pan source and coord-maps correctly (tested), and the Hand/Space tool is U-004
  while touch/pinch is explicitly U-012 scope. *Fix:* wire Space-hold pan (and defer two-finger
  to U-012) when the Hand tool lands; no engine change needed.
- **F-2 (LOW) — `vite.config.ts` hardcodes `docs/acceptance/U-003/**` in `test.include`.**
  Transparent and safe today, but each future unit must hand-edit this list, and it is easy to
  forget. *Fix:* switch to a glob (`docs/acceptance/**/*.test.{ts,tsx}`) once later units'
  target modules exist, or drive it from state.json in the harness.
- **F-3 (LOW) — coverage denominator scoped to `src/core/**`** (inherited F-2 from U-001):
  `renderer.ts`/`overlays.ts`/`CanvasStage.tsx` aren't counted, though they are exercised by the
  browser suite. *Fix:* widen coverage `include` to `src/platform` in U-013's perf/a11y pass.
- **F-4 (LOW) — incremental dirty-rect repaint at fractional zoom (<1).** `paintDisplayRegion`
  clears/blits with fractional device coords, which can leave a sub-pixel seam at a repainted
  sub-rect edge when zoomed out below 1×. Cosmetic only, invisible at integer zooms and in
  fit-to-screen; not a spec violation. *Fix (optional):* expand the cleared/blitted rect by 1
  device px, or round to device pixels, in the fractional-zoom path.
