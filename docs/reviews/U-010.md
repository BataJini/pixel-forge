## Pass 1 — Spec conformance (§3.8 / §5 / criteria.md)

Machine-checkable:
1. **`packFrames` meta rects** — one rect per frame, `w/h == frame size`, within
   bounds, non-overlapping, spacing == `padding`, outer `margin`, per-frame
   `duration` carried. **MET** (held-out + builder suites; `blitOver` clips to the
   cell box so non-overlap holds unconditionally).
2. **Grid / horizontal / vertical atlas dims** — `columnsFor` + geometry produce
   the exact expected dimensions. **MET.**
3. **GIF encode** — bytes begin `GIF89a`; independent hand-rolled `parseGifInfo`
   decodes `frameCount == frames` and correct logical `w/h`. **MET.** Verified the
   `gifenc` global-color-table reuse: with `auto:true`, frame 0 writes the global
   table (`d = Boolean(palette) && !first` → false, no local table) and frames 1+
   with `palette: undefined` reuse it → **no color drift** on the exact-palette path.
4. **Effect-free** — exporters consume `compositeFrame`/`composite` buffers only,
   never the screen; determinism test (`a===b`) passes. **MET by construction**
   (see L-2 re: the literal CRT-toggle machine assertion).
5. **512×512 without freezing (worker)** — `gifWorker.ts` runs the pure encoder off
   the main thread (buffers copied then transferred; progress streamed), with a
   main-thread fallback. **MET (infrastructure present; stress is a QA item).**

Manual/QA-facing:
- **Export GIF dialog** — scale (shared PNG scale ladder), loop default infinite
  (`loopForever` default `true`), frame durations (`delayMs`), Web Worker encode,
  progress toast (`Encoding GIF… done/total`). **MET** (see L-1 re: loop-count UI).
- **Export Spritesheet dialog** — layout / padding / margin / optional POT; saves
  PNG then companion `.json`; atlas slices back to frames (`sliceCel` round-trip
  test). **MET.**

Design conformance: the new `export-dialog.css` uses the bespoke Forge token/bevel
system (`--bevel-raised/inset`, `--focus-ring`, `--px` grid, `--c-*` tokens),
`steps()`-friendly, **no** border-radius / blur / soft gradient / cubic-bezier.
Dialog composes the U-002 `Dialog`/`Button` primitives. **Not templated — MET.**

## Findings

### MEDIUM (advisory — verification integrity / process; owner: manager)
- **M-1 Held-out suite landed in the implementation commit.**
  `docs/acceptance/U-010/export.acceptance.test.ts` was added in the same commit
  `9f2f6f5` as the builder code (commit msg: "previously-missing held-out
  acceptance suite"; "manually integrated to bypass the process-churn"). Unlike
  U-007 (separate `a74364d` test commit), there is no evidence the held-out suite
  pre-dated/constrained the build, weakening the objective-gate independence
  guarantee for this unit (same class as U-008 M-1). *Mitigation:* the test content
  is rigorous and spec-faithful (imports the exact §5 paths; asserts counts, dims,
  durations, non-overlap, GIF89a + decoded frame count), and this reviewer
  **independently** confirmed correctness (deep read + reasoning about `gifenc`
  global-table reuse + full test/build re-runs), so correctness risk is low.
  *Fix:* manager confirms the suite matches Architect intent and treats it as
  authoritative going forward; consider a policy that held-out tests are committed
  before the builder worktree is cut.
- **M-2 Missing committed audit trail; `state.json` pre-marked `verified`.**
  At review time U-010 had no `docs/reviews/U-010.md` or `docs/qa/U-010.md` and only
  an *untracked* `docs/gate/U-010.json`, yet `state.json` already reads
  `"status": "verified"`. The commit message admits manual integration to bypass the
  autonomous loop. Process/audit gap, not a code defect. *Fix:* commit the gate +
  this review + a QA doc; ensure state transitions follow the verify gate. (This
  review supplies the missing `docs/reviews/U-010.md`.)

### LOW (advisory)
- **L-1 GIF loop UI is forever/once only.** The dialog exposes a "loops forever"
  checkbox (`loop` 0 or 1); spec §3.8 says "loop count (default infinite)". The core
  encoder already supports arbitrary `n`. *Fix:* expose a finite loop-count field in
  the U-012/U-013 export polish (core is ready).
- **L-2 Misleading test comment + no GIF-pixel decode test.** `gif.test.ts:95-96`
  states pixels are "verified byte-for-byte in the Browser-Mode test," but
  `export.browser.test.tsx` decodes only PNG/SVG — no test decodes GIF pixels, and
  the "CRT ON vs OFF identical" criterion is exercised for PNG only. Color fidelity
  of the exact path is sound by reasoning, not by a decoded-pixel assertion. *Fix:*
  add a Browser-Mode test that decodes an encoded GIF (and a GIF/spritesheet
  CRT-toggle case) in U-013, or correct the comment.
- **L-3 `alphaThreshold`/`maxColors` not surfaced by the UI export path.**
  `exportGifFile` passes only scale/loop/fps; core defaults (128 / 256) always
  apply. Fine for pixel art; note for completeness.
