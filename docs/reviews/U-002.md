## Pass 2 — Engineering / correctness / reward-hack

- **Module boundaries (constitution)** — clean. Pure/DOM-free logic (`contrast.ts`, `crt.ts`,
  `themes.ts`) is unit-tested headless; React UI in `src/ui/**`; WebAudio glue isolated in
  `src/platform/audio.ts`. No engine logic leaked into UI.
- **Immutability** — honored. Token tables are `as const`/`Readonly`; `ThemeProvider` updates
  state functionally; `cx` is pure. No shared-object mutation.
- **Error handling / client envelope** — `audio.ts` returns the `{ok}|{ok,error}` envelope and
  never throws across the boundary; a muted call is a successful no-op. `localStorage` read/write
  are try/caught (private-mode safe). `useTheme` throws only outside a provider (programmer error).
- **Security** — no `eval`, no `innerHTML`/`dangerouslySetInnerHTML`, no secrets, no runtime
  network; the dither `data:` SVG is static (no user input). No auth/payments/file/crypto surface,
  so no security-reviewer delegation warranted.
- **KISS/DRY/size** — components are 22–95 LoC, single-responsibility; `cx` dedups class joining;
  tokens have one source of truth (TS + CSS with a drift test). No dead code, no `console`/`debugger`,
  no `@ts-ignore`. The two `biome-ignore`s are justified (native `<dialog>` handles Esc so the
  backdrop-click needs no key handler; `!important` is required for the reduced-motion override).
- **Reward-hack scan — CLEAN.** U-002's only held-out artifact is `criteria.md` (untouched — the
  builder wrote its own tests, which is correct here). No `.skip`/`.only`/`xit`/`todo`, no `xfail`.
  The modified tracked test files (`App.browser.test.tsx`, `scaffold.test.ts`, `core/index.test.ts`,

  new `.gitattributes`; no assertions weakened or deleted. Token tests hardcode expected hexes —
  legitimate (that IS the contract) — while contrast tests compute ratios through the real WCAG
  function rather than asserting precomputed numbers. Browser tests use real Chromium + real CSSOM
  (no over-mocking); the audio test correctly runs headless to prove the muted/unsupported paths.

## Design conformance

Genuinely **bespoke**, not templated/stock-kit. The showcase renders the invented Arcade-CRT-over-Forge
system: a hand-drawn on-grid SVG anvil + sparks in a recessed "anvil well" over the theme-independent
`#C8C8C8`/`#8F8F8F` checkerboard (the signature moment), bevelled bento panels, Press Start 2P wordmark,
VT323 status readout, notched `clip-path` Frame, dither surfaces, and live theme/CRT/sound controls.
All five design laws hold — hard edges, stepped tones, zero AA on chrome, integer offsets, one top-left
light source. State is never hue-only (active = Ember face + pressed bevel + `aria-pressed`; danger =
Warning bevel edge + steel text, not red-on-red). No NES.css/98.css lift, no generic SaaS look, no reused
house style. **No design HIGH finding.**

## Findings (all advisory — none block)

- **F-1 (MEDIUM) — `--c-ash` secondary text on `--c-iron` panels fails AA in the *Forge* theme
  (3.79:1 < 4.5).** `ash` is spec'd as "secondary text **on Anvil**" (ash/anvil = 4.79 passes), but
  the showcase applies it over iron panel surfaces in `.pf-tagline`, `.pf-hint`, `.pf-swatches__name`,
  and `.pf-slider__value`. In Forge those render at **3.79:1** (real 14px/10px body text → AA fail);
  the default Arcade theme passes the same pairs (5.14:1), and the enumerated criterion-3 pairs all
  pass, so this is beyond the stated contract but is a genuine WCAG 2.2 AA gap the constitution forbids.
  *Fix:* use `--c-steel` for secondary text on iron, or place secondary text on anvil-colored areas,
  or nudge Forge `--c-ash`/`--c-iron` so ash/iron ≥ 4.5 — and add an ash-on-iron assertion so U-012's
  real workbench can't reintroduce it. (ash/ironHi also computes < 4.5, but no ash text is actually
  placed on iron-hi today — headers use steel/flame — so that pair is not a live defect.)
- **F-2 (LOW) — CRT overlay sits *below* native `<dialog>` top-layer.** `.pf-crt` is a normal
  `z-index:9000` element, but `showModal()` promotes the dialog (and its `::backdrop`) to the top
  layer, so open modals aren't covered by scanlines/glow — a minor deviation from "CRT composites
  above all content," not a functional bug. *Fix (optional):* accept clean modals, or render the CRT
  layer in the top layer / re-mount it above dialogs.
- **F-3 (LOW) — persistent `will-change: transform`** on `.pf-crt__sweep` and `.pf-marquee` keeps a
  compositor layer alive even when idle or at CRT Off. *Fix:* scope `will-change` to the active state
  only, or drop it (the sweep animates transform regardless).
- **F-4 (LOW) — coverage denominator still scoped to `src/core/**`** (inherited U-001 F-2), so the new
  `src/ui`/`src/platform` code isn't counted despite being exercised by 69 passing tests. *Fix:* widen
  the coverage `include` in U-013's perf/a11y pass. Also note: `criteria.md` line 35 names the focus
  ring hex `#FFB03A` (the Forge spark) — the implementation correctly uses the theme `--c-spark` token
  (`#FF2E88` in the Arcade default), which is more spec-aligned than hardcoding; no change needed.
