## Prior blockers — all RESOLVED in this build

- **Prior Review F-1 (undo/redo data corruption) — RESOLVED.** The rewrite uses
  **copy-on-write**: `LayerStack.beginStroke()` (`state/layerStore.ts:378`) clones
  the active layer's buffer and swaps in a *new* layer object + array before any
  in-place `setPixelMut`, and `history.applyPatch` (`core/history.ts:85`) clones
  and returns a new buffer. Structural/metadata snapshots (`commit`, L455) share
  buffers by reference but those buffers are **never mutated after capture**, so a
  structural undo can no longer restore a buffer the draw path later scribbled on.
  Locked in by three real regression tests, incl. the exact prior repro
  (`layerStore.test.ts:214` "interleaved structural op + draw + undo-across",
  `:242` hide+draw+undo-across, `:255` copy-on-write) — all pass.
- **Prior Review F-2 (layer lock not enforced) — RESOLVED.** `deleteLayer`
  (`:238`), `mergeDown` (`:346`) and `flatten` (`:362`) now refuse to touch a
  locked layer; `beginStroke`/`paint` (`:381`,`:399`) no-op on a locked active
  layer; new `activeLocked`/`anyLocked` snapshot flags fold lock state into
  `canDelete`/`canMergeDown`/`canFlatten` so the panel buttons disable in lockstep.
  Covered by `layerStore.test.ts:272-328` (delete/merge/flatten/paint all guarded).
- **Prior Review F-3 / QA F-1 (panel keyboard-inoperable, WCAG 2.1.1 A) — RESOLVED.**
  `CanvasStage.attachKeyboard` now bails on interactive targets
  (`INTERACTIVE_SELECTOR`, `CanvasStage.tsx:112`,`:266`) **before** the
  Enter/Space `preventDefault`, so it no longer hijacks native activation of panel
  buttons. Verified in real Chromium (`LayersKeyboard.browser.test.tsx`: Enter on
  "Add" adds a layer; Space on the eye toggles it). Contrarian check: the canvas
  well and `<canvas>` elements carry **no** `tabindex`/`role`, so they are *not*
  matched by the selector — canvas keyboard drawing (Enter-stamp/Space-pan/
  arrow-nudge/tool-letters) is preserved.
- **Prior Review F-4 (emoji eye/lock glyphs) — RESOLVED.** Eye/lock are now
  bespoke CSS pixel glyphs (`.pf-eye`/`.pf-lock`, state by shape: filled square vs
  hollow ring, solid bar vs outline — not hue). (Grip/nudge glyphs remain unicode →
  L-1 below.)

  acceptance criterion ("every layer op undoable") is met at the store level and the
  code comments acknowledge U-012 unifies the timeline — but until then Ctrl+Z will
  not undo a layer op. *Fold into the U-012 history unification.*
