## Pass 1 — Spec conformance (criteria.md → status)

Machine-checkable (Vitest):
1. **Adding a layer adds an aligned layer to EVERY frame** — MET. `addLayerToAllFrames` mints a fresh transparent buffer per frame with the SAME aligned id/metadata; `FrameStack.addLayer` records it as one undo entry. Tests assert distinct buffer instances (painting one never leaks) + `framesAligned`.
2. **add / duplicate(deep-copy) / delete(last-frame guard) / reorder** — MET. `duplicateFrame` deep-copies every layer buffer and inserts after the source; `deleteFrame`/`canDeleteFrame` guard the last frame; `moveFrame` clamps. Deep-copy independence asserted at both core and store level (editing the copy leaves the source pixel `[0,0,0,0]`).
3. **Playback order + timing from per-frame `durationMs` / global `fps`** — MET. `effectiveDurationMs` (frame duration else fps fallback), `buildTimeline` (cumulative offsets + total), `frameIndexAtTime` (loop/ping-pong/clamp), `playbackOrder` (period 2n-2, no endpoint repeat). Boundary cases tested (t=100→frame 1, wrap at total, negative clamps).
4. **Onion selection = only N prev + N next, never current, clamped at ends** — MET. `selectOnionFrames`/`onionSkinIndices` exclude current, no wraparound, clamp at 0 / n-1, cap at `MAX_ONION_RANGE`, empty on out-of-range current.

Manual / QA:
- **Timeline UI** (add/duplicate/delete/reorder, per-frame duration input, FPS slider) — MET (TimelinePanel + FrameThumb; drag reorder + keyboard move-earlier/later buttons; browser + e2e cover each).
- **Play/pause/stop/loop (+ ping-pong) animate the composite live** — MET (`useFramePlayback` rAF clock maps elapsed→`frameIndexAtTime`; e2e "play advances, pause halts").
- **Onion ghosts: previous warm UNDER next cool, reduced opacity, toggle + range configurable, disabled cleanly** — MET. `onionGhosts` tints prev `warm`/next `cool` with linear falloff, ordered farthest→nearest so nearer paints on top; warm emitted before cool so cool composites above (previous "under" next). `OnionGhosts` blits onto a display-only overlay canvas with `globalAlpha`; ghosts hidden while playing and removed from the DOM when off. Browser test asserts a non-transparent ghost pixel where frame 1 was painted, and its removal on toggle.

### M-1 (MEDIUM, verification-integrity — NOT builder-fixable; owner: manager/Architect)
`docs/acceptance/U-008/` contains only `criteria.md` — there is **no held-out `*.acceptance.test.ts`** (unlike U-003/U-004/U-006/U-007/U-009, which each ship one). The builder's added `docs/acceptance/U-008/**/*.test.{ts,tsx}` include glob therefore matches **zero** files, so the objective gate's "held-out acceptance tests pass" is **vacuously** satisfied for this unit — the only frame tests that ran are builder-authored (and thus builder-editable). The builder acted correctly (must not create files in the protected dir), and their tests + my independent unit/browser/e2e runs do cover all four machine-checkable criteria, so correctness risk is low. **Fix:** the manager/Architect should author `docs/acceptance/U-008/frames.acceptance.test.ts` (importing ONLY from `src/core/frames.ts` per the module-boundary rule — `addLayerToAllFrames`, `duplicateFrame`, `deleteFrame`/`canDeleteFrame`, `moveFrame`, `buildTimeline`/`frameIndexAtTime`, `selectOnionFrames`) before the U-013 global acceptance check, closing the held-out gap.

### L-1 (LOW, history byte accounting)
Structural undo entries under-weight retained memory: `addFrame`/`duplicateFrame` record `256 + frameBytes()` (one buffer's worth) though they create `layerCount` buffers, and `addLayer` records `256 + frameBytes()` though it mints `frameCount` buffers. On a 512×512 canvas with many frames/layers the §6 ~64 MB total-byte cap can be overshot before oldest-eviction triggers. **Fix:** weight frame ops by `frameBytes * layerCount` and `addLayer` by `frameBytes * frameCount`.

### L-2 (LOW, cosmetic playback edge)
Non-loop playback with ping-pong ends by parking on frame `frames.length - 1` (`useFramePlayback` tick), whereas the ping-pong cycle's final visited frame is an interior index. Spec doesn't define the resting frame, so this is cosmetic. **Fix (optional):** land on `frameIndexAtTime(frames, total, { loop:false, pingPong:true })` for consistency.
