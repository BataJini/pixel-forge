## PASS 2 ‚Äî Engineering & reward-hack scan

### Reward-hacking / verification-integrity
- **H-1 (blocking).** The authoritative held-out `shell.acceptance.test.ts`
  (master-spec ¬ß3.7 contract) imports `fuzzyRank` from `src/ui/shell/fuzzy.ts`;
  the module exports `fuzzyFilter` / `fuzzyMatch` / `fuzzyScore` and `index.ts`
  re-exports only those three. `fuzzyRank` exists **nowhere** in `-887-3/src`.
  Objective gate `testExit=1`. `fuzzyFilter(query, items, key)` has the exact
  signature and `{item,score,indices}[]` shape the test needs ‚Äî the alias probe
  passes 5/5 ‚Äî so this is a pure **API-surface naming miss**, not a logic defect.
- **H-2 (blocking, masking).** `vite.config.ts` `test.include` lists U-003..U-009
  and U-011 but **omits U-010, U-012, U-013**. The config's own comment says each
  unit "ACTIVATES its own held-out suite here once the modules it targets exist";
  the builder did not add the U-012 line. Net effect: `npm test` (and the gate's
  `npm test`) never activates the U-012 held-out glob, so a green 673-test run
  gives false assurance and masks H-1. `master`'s config includes all units.
- **No** deleted/weakened assertions, `skip`/`only`/`xit`/`todo`, over-mocking, or
  hardcoded expected values were found anywhere in `src/` or `e2e/`. The
  `e2e/*.spec.ts` edits to sibling suites only add a `dismissWelcome()` call
  (legitimately required now that the shell shows a first-run dialog) ‚Äî not
  weakening. `src/core/tools.ts` additions are pure/immutable.

### Correctness / a11y / security
- `fuzzy.ts` ‚Äî pure, DOM-free, immutable, memoized subsequence scorer with
  word-boundary/consecutive bonuses; correct. The strongest part of the unit.
- `commands.ts` single-catalog pattern structurally guarantees menu‚Üîpalette‚Üîkeyboard
  cannot drift; `validateCatalog` is a real "no dead menu item" guarantee.
- `AppShell` wires **every** action to a real controller/store method ‚Äî no no-op
  stubs; persistence (autosave + gallery + Save) is genuine via `ShellStore`.
- `useShellKeyboard`: single window-level dispatcher with a sane guard order

  guard, then bare chords). Minor: the `Plus`/`Minus` branch at lines ~105-114 is
  dead (falls through to the switch) ‚Üí **L-1** KISS cleanup.
- Security: U-012 adds **no** new untrusted-input parsing (image import /
  deserialize are U-011, unchanged), no `eval`, no secrets, no runtime network.
  Not implicated; no security-reviewer escalation required for this diff.
- Immutability / constitution: honored (new buffers from flip/rotate; catalog
