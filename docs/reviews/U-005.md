## 2. Engineering pass

- **Purity / boundaries:** `core/palette.ts` + `core/color.ts` (HSV) are pure,
  DOM-free, deterministic (FNV-1a content-hash ids; no `Date`/random). Pure state in
  `src/state/colorStore.ts`; React in `src/ui`; browser glue in `src/platform`.
  Matches the constitution's module map.
- **Immutability:** swatch editors copy-before-edit; `paletteSwap`/`snapBuffer*`
  clone then mutate the clone (returns a NEW buffer); `pushRecent` returns a new
  array; `effectivePaintColor` always returns a fresh tuple. Built-ins are now
  **deep-frozen** (colors, tuples, object) — the L-1 latent-mutation risk is closed
  and asserted (`palette.test.ts` "deep-freezes every built-in").
- **Security (input boundaries):** palette import caps input at 5 MB and colors at
  256, strips control chars + length-caps imported `.gpl` names, returns the
  `Result<T>` envelope (never throws) on malformed/empty/oversized input;
  `recentColors` localStorage is fully guarded (private-mode/quota/corrupt) and
  re-validates every entry; `downloadText` uses a Blob + `rel=noopener`; the file
  `<input>` is `accept`-scoped. No `eval`, no `innerHTML`, no secrets, no runtime
  network. Small, defensively-handled surface — no security finding.
- **A11y:** picker fields `role="slider"` + arrow/Shift keys + aria text; swatches
  are labelled buttons with `aria-pressed` and a keyboard bg path; import status is
  `role="status" aria-live="polite"`; visible Spark focus ring (verified in QA
  artifacts). State is conveyed by value + label, never hue alone.

No correctness bug found in any shipped path. The indexed effect ordering
(one-time setup → indexed quantize), StrictMode remount handling
(`appliedPaletteRef` reset), and the exact-match palette-swap over already-quantized
pixels are all correct. **Engineering pass = PASS.**

## 3. Findings (all advisory; none block)

### M-A (MEDIUM — advisory, handoff/integrity) The on-disk objective gate + QA docs point at older worktrees than the build under review
- **Where:** `docs/gate/U-005.json` references `wf_023eceaa-423-24` (the **failed**
  pre-fix build — its evidence shows 195 tests and no indexed-enforcement); `docs/qa/
  U-005.md` references `-35`; the current complete build is **`-43`** (adds the
  `CanvasStage` self-enforcing lock, deep-freeze, and an extra browser test that
  never ran in `-24`/`-35`).
- **Why it matters:** the harness's contract is "pass ONLY if Reviewer = PASS **and**
  QA = PASS **and** the objective gate is clean" — those three must agree on the
  **same** worktree. Right now the persisted gate artifact is from a build that
  predates the H-1 fix entirely.
- **Mitigation already in hand:** I independently re-ran the full objective gate on
  `-43` (build 0, test 0 / 204, held-out 8/8, browser 24, E2E 4/4, lint 0, core
  coverage 97%), so the objective evidence for `-43` exists in §0 above; risk is low.
- **Fix:** integrate `-43` (the superset), and have the independent Verifier
  regenerate `docs/gate/U-005.json` (and reconcile the QA doc) against the exact
  worktree being merged, so the on-disk artifacts match the integrated code.

### L-A (LOW — advisory, carry-over) Coverage `include` is scoped to `src/core/**`
- **Where:** `vite.config.ts:44` (`coverage.include: ['src/core/**']`).
- **What:** `src/state/colorStore.ts` (which hosts `effectivePaintColor`, the
  enforcement heart), `src/platform/{files,recentColors}.ts`, and `src/ui/color/**`
  are exercised by unit + browser + E2E tests but are **not counted** in the 80% line
  gate. The constitution only mandates 80%+ on `src/core/`, which is met at **97%+**,
  so this is not a violation — the same carry-over noted in U-001 F-2 / U-002 F-4.
- **Fix:** widen the coverage `include` to `state`/`platform`/`ui` in a later unit.

### L-B (LOW — advisory, hardening) `readTextFile` reads the whole file before the 5 MB cap applies
- **Where:** `src/platform/files.ts:29-36` (`readTextFile`) vs `palette.ts:275`
  (`MAX_INPUT_BYTES` length check, which runs only after the full read).
- **What:** a pathologically large user-selected file is fully read into a string
  before `parsePalette` rejects it. Low risk (local, user-initiated, no remote
  vector), but defensive parsing should fail fast.
- **Fix:** check `file.size > MAX_INPUT_BYTES` in `onImportFile` / `readTextFile`
  before reading, returning the friendly error without loading the payload.
