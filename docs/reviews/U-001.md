### Design conformance (design-direction.md + constitution) — **PASS**

The scaffold ships a deliberately minimal, **brand-safe** shell plus the token
layer, deferring the full chrome to U-002 (its stated scope) — and it does so
without any forbidden foundation:

- **Tokens match the brief verbatim.** `src/styles/tokens.css` encodes the **Arcade
  CRT** ramp on `:root` (default theme: `--c-anvil #06070c`, `--c-ember #00f0ff`,
  `--c-spark #ff2e88`, `--c-flame #ffd300`, `--c-quench #39ff14`, …) and the
  **Forge** ramp on `[data-theme="forge"]` (`--c-ember #ff6a1a`, `--c-anvil #12100e`,
  …), same token *names* across themes — exactly as design-direction specifies.
  Theme-independent checkerboard tokens present.
- **No forbidden techniques.** No `border-radius`/blur/soft-gradient/`cubic-bezier`
  on chrome. Bevels are hard (`0` blur, `0` spread, whole-`--px` offsets); motion
  utilities are `steps()`; `-webkit-font-smoothing: none`; `--px` grid; visible
  Spark focus ring; `prefers-reduced-motion` reset (the one documented
  `!important`, WCAG 2.2 §2.3.3 justified).
- **No forbidden fonts.** The prior build's banned **Inter** (+ Space Grotesk /
  JetBrains Mono) and `plasma` purple are **gone**. A neutral `ui-monospace` stack
  is a labelled placeholder; the real self-hosted pixel faces (Press Start 2P,
  Pixelify Sans, Silkscreen, VT323) are correctly deferred to U-002.
- **Bespoke, not templated.** `theme-color #06070C` is the brand anvil token; the
  favicon is a hand-drawn 16×16 pixel anvil + spark in brand colors; the shell is a
  single ironed plate with an ember ingot accent — no marketing landing page, no
  outbound template links, no stock-kit look.

Not a generic/templated look → **no HIGH design finding.** (Prior HIGH resolved.)

## Findings (severity-ranked)

### MEDIUM — advisory (does not block)
- **F-1 — CI/deploy wired to `main`, but the repo's default branch is `master`.**
  `.github/workflows/ci.yml` triggers on `push: branches: [main]` and deploys
  `if: github.ref == 'refs/heads/main'`. The worktrees and integration branch are
  `master`, so as delivered the pipeline would never fire on the branch that
  actually receives commits — the unit's "CI on push + static deploy" intent is
  inert until reconciled. Criterion #8 (workflow file exists + base configured)
  still passes, hence advisory.
  - *Fix:* rename the integration branch to `main` on first push, or change the
    workflow trigger/deploy guard to `master` (and keep `README`'s "deploys from
    `main`" note in sync). Confirm before the first real deploy.

### LOW — advisory
- **F-2 — Coverage scope is `src/core/**` only.** Correct per the constitution's
  "80% on `src/core`", so `ui/state/platform` are not coverage-gated. Fine while
  those barrels are trivial; revisit the `include` when they gain real logic so the
  80% floor tracks them too.
- **F-3 — Default `npm test` runs in the Node environment.** DOM/component behavior
  is intentionally covered by the separate `test:browser` (real Chromium) and
  `test:e2e` runs, both green. Well-documented; ensure CI keeps running all three
  (it does). No action needed now.
