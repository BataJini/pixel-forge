## Findings

### F-1 ŌĆö LOW (advisory): Pencil "Alt = temporary eyedropper" not implemented
master-spec ┬¦3.2 lists, for the Pencil, "Alt = temporary eyedropper." In `ToolSession.pointerDown` the pencil path (`beginFreehand`) ignores `mods.alt`, so Alt-drag with the pencil paints instead of sampling. This modifier is **not** in the authoritative `criteria.md` manual list (the Eyedropper tool and its AltŌåÆbg are implemented and tested), so it does not block, but it is a genuine ┬¦3.2 detail.
- **Failure scenario:** With Pencil active, Alt+click paints fg at the point instead of picking the color under the cursor into fg.
- **Fix:** In the pencil branch, when `mods.alt` is set, sample-into-fg (reuse `pickColor`) instead of beginning a stroke ŌĆö or record an ADR folding this convenience modifier into U-012's full keyboard/modifier map.
- File: `src/state/toolSession.ts` (`pointerDown` `case 'pencil'`, ~line 371).

### F-2 ŌĆö LOW (advisory): whole-layer Move/nudge does a full-canvas repaint per event (and edge-destructive nudge)
`previewMove` / `nudge` (whole-layer branch) call `translateBuffer` (full-buffer alloc) + `target.setComposite` (full static repaint) on every pointer-move / arrow press. A full-layer translate genuinely dirties the whole canvas, but the per-event reallocation is avoidable, and at the 512┬▓ ceiling this is heavier than the dirty-rect-within-a-frame budget implies. Arrow-nudge of the whole layer is also destructive at the edges (clipped pixels are lost per press, unlike the drag path which translates from a fixed base). Preview-only; carry to the U-013 performance pass.
- **Fix:** translate/composite only the affected region (or reuse a fixed base for nudge and commit once), and prefer the floating-selection path for constrained moves. Fold into U-013.
- Files: `src/state/toolSession.ts` (`previewMove` ~650, `nudge` ~444).

### F-3 ŌĆö LOW (advisory): preview chrome secondary text uses `--c-ash` on `--c-iron`
`.pf-stage__status` / `.pf-field` (`src/ui/CanvasStage.css:131,181`) put `--c-ash` on `--c-iron` ŌĆö the same ash-on-iron AA gap already tracked as U-002 F-1 (3.79:1 in Forge). Preview-only chrome that U-012 replaces; fold into the existing U-002 F-1 remediation (use `--c-steel` on iron, or add an ash-on-iron contrast assertion).

### F-4 ŌĆö LOW (advisory): coverage `include` still scoped to `src/core/**`; `toolSession.ts` (src/state) isn't counted
The core coverage gate (97.1% stmts) does not include `src/state/toolSession.ts`, the unit's largest new module. It is nonetheless genuinely exercised (89 unit assertions + 6 Browser-Mode assertions), so this is a metric-scope gap, not a testing gap.
