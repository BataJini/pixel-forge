## 2. Engineering pass (re-derived, not inherited)

**Correct and idiomatic.** Verified independently:
- `applyPatch` is genuinely bounds-safe: horizontal clip via `startX/endX/srcCol`, vertical
  clip via the `dy` skip; a rect entirely outside the target returns an unchanged clone
  (held-out bounds-safe test). Patch byte arrays are **copied** at `makePatch` time
  (`extractRectBytes`), so there is no aliasing between a stored patch and the live
  in-place-mutated buffer — undo/redo stays exact.
- `capByBudget` walks newest→oldest, always retains the newest entry even if it alone
  exceeds the byte budget, and caps depth at `maxDepth`. Inverses in `applyListEdit`
  (insert↔remove, move↔reverse-move, replace↔replace) round-trip correctly.
- **Verified a suspected lost-undo bug and cleared it:** `paste()` and
  `commitFloatingSelection()` both commit a pre-existing float through the private
  `commitFloating()`, which **does** record a patch from `floatBase`→live buffer. So
  chaining paste-over-paste does not silently drop an undo entry. (False alarm — no finding.)
- **Single-settle-per-gesture** is the right model for "drag = one entry"; a `floatMove`
  gesture correctly records nothing on `pointerUp` and defers to `commitFloating` on
  tool-change/Enter.

No new blocking defect found. Typecheck, lint, and coverage are clean; no debug statements,
no `.only`/`.skip`/`xit`, no weakened or hardcoded assertions.

## 3. Findings (all advisory — none block)

### N-1 (NOTE — spec bookkeeping, non-blocking) · structural-undo criterion is only partially exercisable at U-006
- The held-out `criteria.md` lists "add/remove/reorder **layer & frame**, resize, paste"
  as undoable. Only **paste/cut/move** are wire-able at U-006; layer/frame/resize live in
  U-007/U-008/U-011. The reversible `applyListEdit` algebra is delivered + round-trip
  tested for them to consume. **Action:** U-007/U-008/U-011 reviews MUST re-verify the
  layer/frame/resize undo criteria against this primitive — do not treat U-006's PASS as
  closing them.

### F-2 (LOW — advisory, now documented) · stack-level coalescing is unsafe for disjoint pixel patches (dead code in U-006)
- `History.record`'s coalesce branch keeps `prev.undo` + `entry.redo`. For `patchEntry`
  commands with **disjoint** dirty rects, `prev.undo` restores only the first sub-step's
  rect, leaving later pixels un-reverted. Not triggered in U-006 (`recordPatch` never sets
  `coalesceKey`; drag→one-entry comes from `settleEdit`). The `-55` build **added the
  contract doc** on `HistoryEntry.coalesceKey` and `patchEntry`. **Action for U-007/U-008:**
  if they want patch coalescing they must union the patches' rects, not swap thunks.

### F-3 (LOW — advisory → U-012) · Ctrl+Z with an uncommitted floating selection reverts twice
- `applyHistoryPatch` first discards an uncommitted float (reverts to `floating.base`) **and
  then** undoes the previous committed entry — so one Ctrl+Z performs two conceptual
  reversions, and the discarded float is not itself redoable. No data loss (clipboard/base
  preserved). Spec doesn't define this edge; fold a decision into U-012's real edit chrome.

### F-4 (LOW — advisory → U-012) · palette/indexed change in the preview is not undoable
- The `CanvasStage` indexed/quantize effect swaps the buffer via `setBuffer` (non-undoable).
  Master-spec §3.6 lists "palette change" among undoable ops, but it is **out of U-006 scope**
  (work-breakdown scopes U-006 to patch/list/caps/coalesce/keys) and the preview is throwaway
  per ADR-010. Wire palette-change history when U-012 assembles the real app; re-verify there.

### A-1 (LOW — advisory → U-013) · bundle size creep
- Production `index` bundle is 279 kB raw / **88.64 kB gz** — under the 250 kB gz budget, but
  trending up. Code-split the Export dialog / exporters when the app shell lands (U-012/U-013).

### A-2 (LOW — advisory → U-012) · the "1px @512² sub-rect" guarantee is proven at engine level, not live
- The live preview is 32×32 (throwaway per ADR-010); the sub-rect memory guarantee is proven
  at the pure-engine level (patch < 1 KB). Re-confirm live when the real 512²-capable workbench
  arrives in U-012.
